Navigation

The navigation in a platform is the one of the key for a great experience. Everyone hates to be lost. Everyone hates to not be able to come back to the right page. A list of navigation patterns is here to help you design your navigation experience, using the suitable class in each situation.

Main tabs

The mains tabs are styled by the .tabsbar class. In this div, a ul list is displayed with tab items in li. The li.selected class handle the selected behavior.

  • Summary
  • Edit
  • Manage

Sub tabs

You may need a second level navigation under a main tab. In addition of the initial .tabsbar class, add a .subtabsBar and you will get the subtab style. In tabs, you can use links or icons in links if needded.

  • Access rights
  • Trash

Modular tabs

Categorize the secondary content by using the modular tabs. Made for small width containers, the modular tabs inherit their style from the .tabsbar class, and contains some specificity with the .modularTabs class in addition.

  • Worklist
  • Clipboard

Back navigation

If you need to add a back button which depends of a filter or search result, display it on top of your page, in the same line than the title. In a .globalActionBar, add your .h1 and your p class .action_bar directoryBack, with your link class .button in it.

Breadcrumb

The breadcrumb is useful for user who needs to navigate in a multi-level tree. Add it on top of your page, usually below the header, to be visible on each page. In a div class .breadcrumb that style the position, a div class .breadcrumbs will style your links.
