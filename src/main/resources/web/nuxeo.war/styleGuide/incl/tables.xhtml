Tables

Tables are commonly used in the platform to display tabular data: in Content Views, Widgets, even in Gadgets in your dashboard. Tables are also used to display forms, but are not visibles in these case. Here is a list of the available CSS classes, with their particularities.

dataList

The most common table in the platform, this class is used in Content Views. 100% width, bottom borders, and a lot of space in rows.

Class Use
.dataList Display a listing

dataTable

A recent addition, since you may need some tables in your widgets. 100% width, with a border all around, and designed to list a lot of data in a small space.

Class Use
.dataTable Simple small table

dataTableLine

As the dataTable, this table model is made for the small boxes, to list many elements, keeping the readability. No borders around, 100% width, and a bold behavior in thead.

Class Use
.dataTableLine Display a listing

Invisible tables

The invisible tables are generally used in forms to structure the content, align labels and fields, without needing to be seen. A .labelColumn is used for the label cells, a .fieldColumn for fields.

  • .dataInput for global forms in a page,
  • .dataTableNoBorder for specific edition form in small space.

Design a table

A list of class are available to help you design your table. Small columns for icons or checkboxes, or tables with fixed column width, constrain your tables for a great display in your layout. You can design your table by using specific class names, related to the content, or by using width, no matter with content it will display.

Content class names

The content class names are a set of class, named by their role, to help your cells have the right behavior. Note that the labelColumn cell is right align in forms, and take back its left align in data tables
