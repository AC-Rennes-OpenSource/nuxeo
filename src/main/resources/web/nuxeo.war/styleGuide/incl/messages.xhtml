Messages

Feedback, status and help messages are used to communicate with users after a task: to confirm that a task has been done, to tell that the platform is processing the task and needs a bit more time, to ask users to do the task one more time as the request couldn't be performed, to show users what actions can be done and where, or to indicate the state of a document. A feedback message is always a good way to show users what happened or what is happening: the content of the message is as important than the presence of the message itself. The fewer surprise there is during a process, the more intuitive the interface will be.

Feedback messages

Feedback messages showing a global status are displayed in the top right corner of the window. Use the .facesStatusMessage and add a secundary class for the type of your message: .infoMessage, .successMessage, .warningMessage, or .errorMessage. If you want to display a feedback message under a field in a form, use only the class of the message type.

Your workspace is created
Your task is achieved
The task will expired today
Invalid password

Status messages

If you need to show a feedback or status message in a modal pop-up, or at the end of a process in a screen, status messages will be helpful to tell users what they did and, most of all, to explain what is happening next.

Define your message with a .processMessage div class, then add a secondary class to display the corresponding icon that will define the type of your message to users. Five classes are available to define a status message: .completeSuccess, .completeError, .completeWarning, .running, and .scheduled.

Your process is complete, you can continue.
Your process contains an error, please relaunch the process.
Your process has a warning state, be careful and don't forget to save.
Your process is running, please be patient.
Your process is scheduled, please don't run another heavy action.

At last, some complementary classes are available to add a detail, make a field more explicit, or help users understand a specific action.

.detail
A detail under a field in a form
.quote
A quote, or a description
.specification
A specification paragraph, the default opening paragraph class
.widgetHelpLabel
